Sentiment Analysis

Characterizing Developers’ Sentiments Using Large Language Models.

Sentiment Analysis (SA) has become an essential technique in Software Engineering (SE) for understanding developers’ emotions, improving collaboration, and enhancing software quality.

Recent advances in Large Language Models (LLMs) have yielded more powerful approaches to natural language–processing tasks, including SA.

In this project, we evaluate the effectiveness of several LLMs at performing sentiment analysis on comments from issue discussions in open‑source GitHub repositories.


Data Extraction

The process begins with the extraction phase, during which comments from GitHub issues are collected—totaling over 1.7 million tokens.

Data Preprocessing

Next, the raw texts are cleaned—removing URLs, code blocks, and other noise—resulting in a reduction of over 40% in the total token count.

Sentiment Classification

Each comment is sent to the LLMs with a prompt that instructs the model to classify it as positive, negative, or neutral.

Results Visualization

Sentiment scores are aggregated to generate concise visual summaries to highlight actionable insights.

Project information

  • Category: Sentiment Analysis
  • Organization: UFS
  • Project date: Sept. 2024 - Present

In our most recent experiment, we selected three projects—Grafana, TensorFlow, and VSCode—and defined a set of recent releases for each. We then classified every comment in the issue threads and grouped the results by release. Using the GitHub API, we extracted data from 1,803 issues and 13,608 comments, totaling 1,749,254 tokens for the TensorFlow project. After data preprocessing, we achieved an average token reduction of 44.20% across all seven releases. Sentiment classification was performed with three models: GPT-4o mini, DeepSeek V3, and Gemini 2.0 Flash. Across all releases, the vast majority of comments were either neutral or positive, indicating a healthy and constructive community.

  • Dominance of neutral discussions, focusing on facts and technical clarifications.
  • Strong presence of positive feedback, highlighting appreciation, collaborative problem-solving, and helpful contributions.
  • Minority of negative sentiment, suggesting that critical feedback remains constructive and limited in scope.
