Oil & Gas

Lithological classification based on oil well logs.

Lithofacies classification is the process of identifying the rock type present at a point in an oil well, based on its properties, known as well logs. Historically, this task has been performed by geologists, but in recent years, machine learning algorithms, such as Random Forest (RF), have been widely used.

My aim in this field is to identify lithological types using conventional well logs (e.g., density, radioactivity, and resistivity) and image logs, employing algorithms such as Random Forest (RF) and Extremely Randomized Trees (ET).

Work in collaboration with a multidisciplinary team of geologists and engineers, including current and former Petrobras employees, to validate the results.


Data Extraction

The process begins with the download automation phase, during which Selenium scripts systematically retrieve petroleum well files from a repository of Brazilian terrestrial sedimentary basins—amassing over 40 TB of data from REATE’s public repository.

Data Preprocessing

Data preprocessing is the most time‑intensive phase, requiring regular consultations with senior geologists and engineers to validate each transformation step.

Classification Performance

Classification performance is best evaluated using a confusion matrix, which shows how each model performs on every lithology class.

Results Visualization

The predicted classes are plotted alongside the actual classes to assess our models’ accuracy.

Project information

  • Category: Lithofacies Classification
  • Organization: UFS
  • Project date: Sept. 2023 - Present

In our most recent comparative analysis of lithology prediction models—from well‑log features to geological units and class balancing—we evaluated three Random Forest approaches against true lithology columns. Model 1, based solely on well‑log data, often missed thinner or less common units; Model 2 augments well‑logs with geological‑unit variables, yielding noticeably better alignment on sandstone, limestone, and shale intervals; and Model 3 builds on Model 2 by applying class balancing, delivering the most accurate match to the true lithology columns.

  • Model 1 (only well‑logs): Predicted layers often miss thinner or less common lithological units.
  • Model 2 (well‑logs + geological units): Yields better alignment between true and predicted sandstone, limestone, and shale intervals.
  • Model 3 (well‑logs + geological units + class balancing): Provides the most accurate match to true lithology columns.
